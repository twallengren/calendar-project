# Contributing

Thank you for your interest in contributing to this project.

## Pull Request Workflow

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Run tests locally: `./gradlew :tools:test`
5. Submit a pull request

## Code Style

- Follow existing code conventions
- Use meaningful variable and method names
- Add tests for new functionality

## Data Contributions

When contributing calendar data:

- Cite authoritative sources for holiday dates
- Include provenance information in comments
- Ensure data accuracy

## Testing

All contributions should include appropriate tests:

- Unit tests for new logic
- Integration tests for new features
- Golden tests for output format changes

## Commit Messages

Use clear, descriptive commit messages:

```
Add Hijri calendar support for Eid dates

- Implement HijrahChronology translation
- Add tests for Hijri date expansion
```
